# Backend Template

Reusable backend starter structure for another software project.

## Suggested Structure

- `src/` application code
- `prisma/` database schema, migrations, and seed scripts
- `scripts/` one-off maintenance or import/export scripts
- `test/` unit and integration tests

## Notes

- Copy this folder as the base for a new backend.
- Add environment variables in `.env.example`.
- Keep sync/integration logic isolated in dedicated modules.
